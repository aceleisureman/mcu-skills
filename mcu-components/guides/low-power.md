# 低功耗设计规范

面向电池供电设备的系统级低功耗设计方法。测量方法见 `debugging-testing.md` 第 6 节，异常耗电排查见 `pitfalls.md` 第 9 节。

## 1. 功耗预算方法

设计前先做功耗预算表，倒推各状态允许电流：

```
电池容量 (mAh) × 0.8(可用率) ≥ 平均电流 (mA) × 目标续航 (h)
平均电流 = Σ(状态电流 × 状态时间占比)
```

示例：2000mAh / 1 年续航 → 平均电流 ≤ 0.18mA → 休眠电流必须 μA 级。

| 状态 | 电流 | 占比 | 贡献 |
|------|------|------|------|
| 深度休眠 | 5μA | 99.4% | 5μA |
| 采集(100ms/次/分钟) | 5mA | 0.17% | 8.3μA |
| 上报(2s/次/小时) | 120mA | 0.056% | 66μA |

结论：无线上报是大头，优化上报频率/时长收益最大。

## 2. MCU 低功耗模式选择

| 模式 | 典型电流 | 保留内容 | 唤醒源 | 适用 |
|------|----------|----------|--------|------|
| Sleep/轻睡 | 1~3mA | 全部 | 任意中断 | 短暂空闲 |
| Stop/深睡 | 2~20μA | RAM+GPIO | EXTI/RTC | 周期采集主力模式 |
| Standby/关断 | 0.3~2μA | 备份域 | 复位/RTC/唤醒脚 | 超长待机、状态可丢 |

- STM32：优先 Stop 模式（RAM 保留，唤醒后接着跑）；唤醒后需重新配置时钟
- ESP32：deep sleep（RTC 内存可存少量状态）；light sleep 保持 WiFi 连接场景用
- 唤醒时间与启动耗电也要算入预算（ESP32 深睡唤醒重启 ~300ms）

## 3. 硬件设计要点

- **分区供电**：传感器/无线模块用 MCU GPIO 或负载开关（SGM2576/AO3401）控制，休眠时彻底断电
- **警惕静态电流**：上拉电阻（10kΩ 上拉在低电平时漏 330μA！休眠前 GPIO 置合适电平）、分压采样电阻（用 MΩ 级 + MOS 开关切断）、LED、LDO 静态电流（选 IQ < 2μA 的 HT7333/RT9013）
- DC-DC 轻载效率：选带 PFM/省电模式的（MP2315 轻载 80%+）
- 未用引脚配置为模拟输入（禁止悬空输入态）
- 调试口（SWD）外部上拉在量产版去掉

## 4. 软件设计要点

- 事件驱动架构：主循环 = 处理事件 → 无事件立即休眠；禁止轮询延时等待
- 外设用完即关：ADC/UART/SPI 时钟门控，无线模块发完立即进 PSM/断电
- 降频：无实时性要求时主频降至满足计算需求的最低档
- RTC 定时唤醒 + 中断唤醒结合；唤醒后批量处理再睡
- 无线优化：合并上报（攒 N 条一起发）、缩短连接保持、NB-IoT 用 PSM/eDRX

## 5. 低功耗检查清单

- [ ] 实测各状态电流与预算表一致（μA 级用功耗分析仪/μCurrent）
- [ ] 休眠电流逐路排查：断开各模块电源确认漏电来源
- [ ] 所有 GPIO 休眠前状态已规划（上拉脚置高、推挽脚无灌流）
- [ ] 传感器全部进入 sleep 命令模式或断电
- [ ] 看门狗与低功耗模式兼容（IWDG 在 Stop 下仍运行会唤醒！按需关闭或喂狗策略适配）
- [ ] 电池电压监测分压电路带开关
- [ ] 上电浪涌不超过电池放电能力（锂亚电池尤其注意，需并联电容）
