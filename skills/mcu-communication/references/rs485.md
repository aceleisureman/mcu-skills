# RS485 通信开发规范

## 1. 概述与选型指南

### 常见型号对比

| 型号 | 供电 | 速率 | 节点数 | 特点 | 价格 |
|------|------|------|--------|------|------|
| MAX485 | 5V | 2.5Mbps | 32 | 经典、廉价、半双工 | ¥1~3 |
| MAX3485 | 3.3V | 10Mbps | 32 | 3.3V版本 | ¥3~6 |
| SP3485 | 3.3V | 10Mbps | 32 | 低功耗、半双工 | ¥2~5 |
| SP485R | 5V/3.3V | 5Mbps | 256 | 高节点数、1/8单位负载 | ¥3~6 |
| MAX13442 | 5V | 16Mbps | 128 | 集成保护、全双工可选 | ¥8~15 |
| ADM2582E | 5V | 16Mbps | 256 | 集成隔离+DC-DC | ¥20~40 |

### 选型建议
- **5V 系统** → MAX485（经典廉价）
- **3.3V 系统** → SP3485 / MAX3485
- **多节点(>32)** → SP485R（256 节点）
- **工业隔离** → ADM2582E（集成隔离和电源）
- **高速通信** → MAX13442

## 2. 硬件设计规范

### 半双工 RS485 电路

```
                    VCC (5V/3.3V)
                      │
                   ┌──┴──┐
              ┌────┤VCC  │
              │    │      │
   MCU TXD ───┤────┤ DI   │──── A线 ───┬──── 总线A
              │    │      │            │
   MCU RXD ───┼────┤ RO   │           120Ω 终端电阻
              │    │      │            │
   MCU DE ────┼────┤ DE   │──── B线 ───┴──── 总线B
   MCU RE ────┘    │ /RE  │
                   │      │
                   │ GND  │── GND
                   └──────┘

   DE 和 /RE 可短接, 用一个GPIO控制方向
   DE=1: 发送模式; DE=0: 接收模式
```

### 关键设计要点

1. **终端电阻**：总线两端各接 120Ω 终端电阻（匹配双绞线特性阻抗）
2. **偏置电阻**：A 上拉 390Ω 到 VCC，B 下拉 390Ω 到 GND（确保空闲时总线为确定状态）
3. **TVS 保护**：A/B 线对地接 TVS 二极管（如 SMAJ12CA）
4. **方向控制**：DE 和 /RE 短接，用一个 GPIO 控制（高=发送，低=接收）
5. **光耦隔离**：工业场景建议使用光耦隔离（6N137）或集成隔离芯片
6. **线缆选择**：使用双绞线（特性阻抗 120Ω），如 RVSP 2×0.5
7. **总线拓扑**：手拉手菊花链布线，避免星形分支

### 电气参数

| 参数 | 值 |
|------|-----|
| 差分输出电压 | 1.5V ~ 5V |
| 接收灵敏度 | ±200mV |
| 最大总线长度 | 1200m（9600bps） |
| 最大节点数 | 32（标准）/ 256（1/8负载） |
| 通信速率 vs 距离 | 10Mbps@12m, 100kbps@1200m |

## 3. 驱动开发规范

### RS485 方向控制驱动

```c
/**
 * @file rs485.c
 * @brief RS485 半双工通信驱动
 */

typedef struct {
    uart_config_t  uart;
    gpio_port_t    de_port;
    gpio_pin_t     de_pin;
    uint32_t       baudrate;
} rs485_handle_t;

void rs485_init(rs485_handle_t *h, uart_config_t *uart,
                gpio_port_t de_port, gpio_pin_t de_pin) {
    h->uart = *uart;
    h->de_port = de_port;
    h->de_pin = de_pin;
    gpio_set_output(de_port, de_pin);
    gpio_write_low(de_port, de_pin);  /* 默认接收模式 */
}

/* 切换到发送模式 */
static void rs485_set_tx(rs485_handle_t *h) {
    gpio_write_high(h->de_port, h->de_pin);
    delay_us(10);  /* 方向切换延时, 等待收发器稳定 */
}

/* 切换到接收模式 */
static void rs485_set_rx(rs485_handle_t *h) {
    /* 关键: 等待最后一个字节完全发送后再切换方向 */
    while (!uart_tx_complete(&h->uart));  /* 等待发送完成 */
    delay_us(10);
    gpio_write_low(h->de_port, h->de_pin);
}

/* 发送数据 */
rs485_status_t rs485_send(rs485_handle_t *h, const uint8_t *data, uint16_t len) {
    rs485_set_tx(h);
    uart_write(&h->uart, data, len);
    rs485_set_rx(h);  /* 发送完成后立即切回接收 */
    return RS485_OK;
}

/* 接收数据 (带超时) */
rs485_status_t rs485_recv(rs485_handle_t *h, uint8_t *buf, uint16_t *len,
                          uint32_t timeout_ms) {
    return uart_read_timeout(&h->uart, buf, len, timeout_ms);
}
```

### Modbus RTU 帧驱动

```c
/**
 * @file modbus_rtu.c
 * @brief Modbus RTU 协议帧处理
 * 
 * RTU 帧格式:
 *   地址(1B) + 功能码(1B) + 数据(NB) + CRC16(2B)
 *   帧间隔: ≥ 3.5 个字符时间
 */

/* Modbus CRC16 (多项式 0xA001) */
uint16_t modbus_crc16(const uint8_t *data, uint16_t len) {
    uint16_t crc = 0xFFFF;
    while (len--) {
        crc ^= *data++;
        for (int i = 0; i < 8; i++) {
            if (crc & 0x0001)
                crc = (crc >> 1) ^ 0xA001;
            else
                crc >>= 1;
        }
    }
    return crc;
}

/* 计算帧间间隔 (3.5 个字符时间, ms) */
uint32_t modbus_frame_gap_ms(uint32_t baudrate) {
    /* 1个字符 = 11 bit (起始+8数据+校验+停止) */
    /* 3.5 字符 = 38.5 bit */
    uint32_t gap = (38500 + baudrate - 1) / baudrate;  /* ms */
    if (baudrate > 19200) gap = 2;  /* >19200 固定 1.75ms */
    return gap;
}

/* 发送 Modbus RTU 请求 */
mb_status_t modbus_read_holding_regs(rs485_handle_t *h, uint8_t slave_addr,
                                     uint16_t reg_addr, uint16_t reg_count,
                                     uint8_t *response, uint8_t *response_len) {
    uint8_t frame[8];
    uint16_t crc;

    /* 组帧: 地址 + 功能码(0x03) + 起始地址 + 寄存器数 + CRC */
    frame[0] = slave_addr;
    frame[1] = 0x03;  /* Read Holding Registers */
    frame[2] = (reg_addr >> 8) & 0xFF;
    frame[3] = reg_addr & 0xFF;
    frame[4] = (reg_count >> 8) & 0xFF;
    frame[5] = reg_count & 0xFF;
    crc = modbus_crc16(frame, 6);
    frame[6] = crc & 0xFF;        /* CRC 低字节在前 */
    frame[7] = (crc >> 8) & 0xFF;

    /* 发送请求 */
    rs485_send(h, frame, 8);

    /* 等待响应 */
    uint16_t recv_len = 0;
    if (rs485_recv(h, response, &recv_len, 1000) != RS485_OK) {
        return MB_ERR_TIMEOUT;
    }

    /* 验证响应 */
    if (recv_len < 5) return MB_ERR_INVALID_FRAME;
    if (response[0] != slave_addr) return MB_ERR_WRONG_SLAVE;
    if (response[1] != 0x03) return MB_ERR_FUNCTION;

    /* CRC 校验 */
    crc = modbus_crc16(response, recv_len - 2);
    if ((response[recv_len-2] != (crc & 0xFF)) ||
        (response[recv_len-1] != (crc >> 8))) {
        return MB_ERR_CRC;
    }

    *response_len = recv_len;
    return MB_OK;
}
```

## 4. 调试与测试规范

### 验证清单
- [ ] A/B 线极性正确（A 接 A，B 接 B）
- [ ] 总线两端 120Ω 终端电阻已接
- [ ] DE/RE 方向控制 GPIO 配置正确
- [ ] 发送后立即切回接收模式
- [ ] 波特率两端一致
- [ ] 偏置电阻（如需要）已接

### 调试工具
- **USB-RS485 转换器**：连接电脑调试
- **Modbus Poll / Modbus Slave**：PC 端 Modbus 调试软件
- **示波器**：差分探头测 A-B 电压波形
- **逻辑分析仪**：抓取 UART 波形分析时序

### 常见测试场景
1. 单主机单从机点对点通信
2. 多从机轮询
3. 长距离通信（测试不同波特率下最大距离）
4. 干扰环境测试（电机旁、变频器旁）

## 5. 常见问题与避坑指南

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 通信完全不通 | A/B 接反 | 对调 A/B 线 |
| 只能发不能收 | DE 未切回低 | 发送完成后立即拉低 DE |
| 接收数据残缺 | 切换方向太快 | 等待 UART 发送完成(TC标志)再切换 |
| 通信距离短 | 无终端电阻 | 总线两端接 120Ω 电阻 |
| 多节点冲突 | 多个设备同时发送 | Modbus 必须单主机轮询 |
| 偶尔通信错误 | 干扰大 | 加 TVS 保护、使用屏蔽双绞线 |
| 总线空闲时乱码 | 无偏置电阻 | 加上下拉偏置电阻(390Ω) |
| 高波特率误码高 | 线缆质量差/距离远 | 降波特率或换更好的线缆 |
| 从机不响应 | 地址错误/波特率不匹配 | 核对从机地址和波特率 |
| Modbus CRC 错误 | 帧不完整 | 确保帧间间隔 ≥ 3.5 字符时间 |
