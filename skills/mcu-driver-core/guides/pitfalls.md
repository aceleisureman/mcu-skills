# 常见问题与避坑指南

本文档按类别整理了 MCU 开发中常见的问题，每个问题包含**现象描述 → 原因分析 → 解决方案**。

---

## 1. 电源类问题

### 1.1 电源纹波过大

**现象**：ADC 读数跳动大，UART 偶尔误码，MCU 随机复位。

**原因**：
- DC-DC 输出纹波过大（> 100mV）
- LDO 输入输出压差不足，进入 Dropout 区
- 去耦电容不足或 ESR 过大
- 电源走线过细，寄生电感导致瞬态响应差

**解决方案**：
- DC-DC 输出加 LC 滤波（10μH + 22μF）
- LDO 输入至少比输出高 0.5V（AMS1117 需 1V 以上）
- 增加 10μF 钽电容 + 100nF 陶瓷电容组合
- 电源走线宽度按电流计算，加宽走线

### 1.2 上电时序错误

**现象**：MCU 无法启动，外设初始化失败，I2C/SPI 设备不响应。

**原因**：
- 外设电源先于 MCU 电源上电，导致 I/O 在 MCU 上电前被驱动
- 多路电源同时上电但稳定时间不同，MCU 启动时外设未就绪
- 上电复位不完整，MCU 在电源稳定前开始执行

**解决方案**：
- 使用带 EN 引脚的稳压器，通过 RC 延时控制上电顺序
- MCU 启动后延时 100~500ms 再初始化外设
- 使用电源监控 IC（如 TPS3850）确保电源稳定后才释放复位
- I/O 使用上拉/下拉电阻确定上电时的已知状态

### 1.3 掉电复位不完全

**现象**：快速断电再上电后，MCU 不启动或外设状态异常。

**原因**：
- 掉电时电源电容放电不完全，电压在阈值附近振荡
- 复位电路响应慢，MCU 在低电压下执行了错误代码
- Brown-out 复位（BOR）未启用或阈值设置不当

**解决方案**：
- 启用 MCU 内部 BOR（Brown-out Reset），设置合适的阈值
- 复位脚加 100nF 电容延长复位时间
- 电源端加大容量电容（100μF 以上），减缓掉电速度
- 软件加掉电检测（ADC 监控电源电压），提前保存数据

### 1.4 LDO 发热严重

**现象**：LDO 稳压器烫手，长时间工作后输出电压下降。

**原因**：
- 功率损耗 P = (Vin - Vout) × I 过大
- PCB 散热铜面积不足
- 负载电流超过 LDO 额定值

**解决方案**：
- 计算功耗：5V→3.3V @ 200mA → P = 0.34W
- 增大 PCB 散热铜面积（TO-252 封装至少 100mm²）
- 改用 DC-DC 降压（效率 90%+，功耗仅 0.07W）
- 选用更大电流规格的 LDO 或分散负载

## 2. 时钟类问题

### 2.1 晶振不起振

**现象**：MCU 不工作，无时钟信号，晶振引脚无波形或波形极小。

**原因**：
- 负载电容不匹配（容值与晶振规格不符）
- 晶振焊接损坏（过热或机械应力）
- 晶振引脚走线过长，寄生电容过大
- MCU 晶振驱动增益设置不当（Gain too low / too high）

**解决方案**：
- 按晶振数据手册选择负载电容：CL = (C1×C2)/(C1+C2) + Cstray
- 焊接温度 < 260°C，时间 < 3 秒
- 晶振紧贴 MCU，走线 < 5mm，晶振下方禁止走其他信号
- 调整 MCU 晶振驱动增益（如 STM32 的 HSE Drive）
- 测试方法：示波器 10X 探头轻触 OSC_OUT 引脚

### 2.2 时钟配置错误

**现象**：UART 波特率偏差大（乱码），定时器周期不对，USB 不识别。

**原因**：
- PLL 倍频/分频计算错误
- 外设时钟预分频设置不当
- 使用内部 RC 振荡器（精度 ±1%~5%），高波特率通信误差超标

**解决方案**：
- 仔细核对时钟树配置，用 CubeMX 等工具验证
- 高精度通信（UART > 115200, USB, CAN）必须用外部晶振
- 内部 RC 校准：部分 MCU 支持软件校准到 ±0.5%
- 测试方法：MCO 引脚输出时钟，用示波器/频率计验证

### 2.3 32.768kHz RTC 晶振问题

**现象**：RTC 走时不准，每天偏差数秒，或 RTC 不工作。

**原因**：
- 32.768kHz 晶振负载电容通常 12.5pF，电容选错
- 晶振抗干扰差，靠近高速信号被干扰
- PCB 焊接残留助焊剂导致漏电

**解决方案**：
- 负载电容按公式计算，通常 6pF~12pF（含寄生电容）
- RTC 晶振远离高速信号，用地线包围保护
- 清洁 PCB 焊接区域，去除助焊剂残留
- 软件校准：定期用网络时间/NTP 校正

## 3. 通信类问题

### 3.1 I2C 地址冲突

**现象**：多个 I2C 设备只识别到一个，或数据错乱。

**原因**：
- 两个器件地址相同
- 器件地址引脚（A0/A1/A2）未正确配置
- 7 位地址与 8 位地址混淆（左移 1 位差异）

**解决方案**：
- 选型时确认地址可配置（ADDR 引脚接 VDD/VSS）
- 使用 I2C 多路复用器（如 TCA9548A）扩展总线
- 明确地址格式：7 位地址 vs 8 位地址（含 R/W 位）
- 使用 I2C 扫描函数确认每个设备地址唯一

### 3.2 SPI 模式错误

**现象**：SPI 通信读到全 0 或全 FF，数据移位。

**原因**：
- CPOL/CPHA 配置与器件要求不匹配
- CS 极性错误（部分器件高有效）
- 位序（MSB/LSB）设置错误
- 时钟频率超过器件最大值

**解决方案**：
- 查器件数据手册确认 SPI Mode (0/1/2/3)
- 用逻辑分析仪抓波形，对比手册时序图
- 确认 CS 低有效（绝大多数器件）还是高有效
- 确认 MSB First（大多数器件）还是 LSB First
- 降低时钟频率测试（先 100kHz 通信成功再提高）

### 3.3 UART 波特率偏差

**现象**：UART 收发乱码，降低波特率后正常。

**原因**：
- MCU 时钟精度不够（内部 RC 振荡器 ±1%~5%）
- 波特率分频计算有舍入误差
- 通信两端波特率不完全一致

**解决方案**：
- 高波特率（> 115200）使用外部晶振（精度 ±20ppm）
- 检查波特率分频器是否有余数，选择误差最小的分频值
- 通信两端波特率误差 < 2.5%（8 位数据 + 1 停止位）
- 使用自动波特率检测（部分 MCU 和模块支持）

### 3.4 I2C 总线挂死

**现象**：I2C 总线 SCL 或 SDA 一直为低，无法通信。

**原因**：
- 从机在接收过程中被复位（未完成 ACK），SDA 保持低电平
- 主机读取最后一个字节时未发送 NACK，从机继续驱动 SDA
- 硬件故障导致线路短路

**解决方案**：
- 软件恢复：SCL 保持输出，发送 9 个时钟脉冲让从机释放 SDA
- 重新发送 Stop 条件复位总线
- MCU I2C 外设软件复位
- 硬件方案：I2C 总线加 Bus Buffer（如 PCA9515）支持隔离

## 4. GPIO 类问题

### 4.1 浮空输入导致电平不确定

**现象**：GPIO 读数随机跳动，MCU 功耗偏高，偶发中断。

**原因**：
- GPIO 配置为浮空输入，引脚电平由环境噪声决定
- 未用引脚悬空，产生漏电流
- 按键未接上拉/下拉电阻

**解决方案**：
- 输入引脚配置为上拉或下拉输入
- 未使用引脚配置为模拟模式（最低功耗）或输出低
- 按键电路：一端接 GPIO（上拉输入），另一端接 GND

### 4.2 引脚复用冲突

**现象**：某外设不工作，或 GPIO 控制无效。

**原因**：
- 同一引脚被两个外设复用（如 PA9 同时用于 UART1_TX 和 TIM1_CH2）
- 引脚的 Alternate Function 映射不正确
- JTAG/SWD 调试引脚未释放

**解决方案**：
- 查阅 MCU 数据手册的引脚复用表（AF table）
- 确认引脚 AF 编号配置正确
- 如需使用 JTAG/SWD 引脚，先调用引脚重映射（Remap）
- 使用 CubeMX / IDE 引脚配置工具避免冲突

### 4.3 GPIO 驱动能力不足

**现象**：GPIO 输出高电平带载后电压下降，LED 不亮或蜂鸣器声音小。

**原因**：
- GPIO 驱动电流超过规格（通常 4~25mA）
- 未加限流电阻，直接驱动 LED
- 驱动容性负载（如长线缆）导致瞬态电流过大

**解决方案**：
- LED 串联限流电阻：R = (V_GPIO - V_LED) / I_LED
- 大电流负载用三极管/MOSFET 驱动
- MCU 推挽输出驱动能力不足时改用开漏 + 上拉（或外部驱动）

## 5. ADC 类问题

### 5.1 参考电压不稳定

**现象**：ADC 读数跳动，测量结果与实际电压有偏差。

**原因**：
- VREF 与 VDD 共用，数字噪声耦合到参考电压
- LDO 输出纹波大
- 参考电压源精度不足

**解决方案**：
- 使用独立 LDO 给 VDDA/VREF 供电
- VDDA 串磁珠隔离 + 1μF + 100nF 去耦
- 高精度场景用外部基准（TL431, REF3133）
- 启用 MCU 内部参考电压校准

### 5.2 输入阻抗不匹配

**现象**：ADC 读数偏小，多通道切换时第一个读数不准。

**原因**：
- 信号源内阻过大，采样电容充电不足
- 多通道扫描时，前一通道的采样电容电荷影响下一通道
- 采样时间设置过短

**解决方案**：
- 信号源内阻 < 10kΩ，否则加运放跟随器
- 增加采样时间（STM32 Sampling Time 设为 239.5 cycles）
- 多通道扫描时增加额外采样时间或插入空闲通道
- ADC 输入加 RC 低通滤波（1kΩ + 10nF），同时降低源阻抗

### 5.3 采样时间不足

**现象**：ADC 读数偏低，温度变化时偏差增大。

**原因**：
- 采样时间 < 采样电容充电时间
- 高阻抗源需要更长充电时间

**解决方案**：
- 采样时间 > (R_src + R_adc) × C_adc × 7.6
- 典型值：R_src=10kΩ, C_adc=5pF → 采样时间 > 0.4μs
- 72MHz ADC 时钟，15 cycles 采样 = 0.21μs（不够），改 71.5 cycles = 1.0μs

## 6. 中断类问题

### 6.1 中断优先级配置错误

**现象**：关键中断被低优先级中断阻塞，实时性差。

**原因**：
- 所有中断配置为相同优先级，无法抢占
- 高优先级中断被低优先级中断延迟
- 中断分组（Preemption/Sub-priority）设置不当

**解决方案**：
- 实时性要求高的中断（如电机控制、安全检测）设为最高优先级
- 通信中断（UART/I2C）中等优先级
- 定时器/低频中断设为低优先级
- 使用抢占优先级（Preemption）允许高优先级中断打断低优先级

### 6.2 中断嵌套过深

**现象**：中断响应延迟大，低优先级中断长时间不执行。

**原因**：
- 高优先级中断频繁打断低优先级中断
- 中断中执行了耗时操作，占用 CPU 时间长

**解决方案**：
- 高优先级 ISR 尽量短（< 10μs），仅设置标志
- 限制中断嵌套层数（通常不超过 2 层）
- 耗时操作放到主循环或低优先级中断中处理
- 使用 DMA 减少中断频率

### 6.3 中断中执行耗时操作

**现象**：其他中断响应延迟，主循环卡顿，UART 丢数据。

**原因**：
- ISR 中调用 `delay_ms()`、`printf()`、大量计算
- ISR 中执行 I2C/SPI 通信（阻塞等待）
- ISR 中有大循环

**解决方案**：
- ISR 中仅：读状态寄存器、清中断标志、设置事件标志
- 所有逻辑处理在主循环或任务中执行
- 如需在中断中通信，使用 DMA + 非阻塞方式
- 必要的延时用定时器中断代替阻塞延时

## 7. 存储类问题

### 7.1 Flash 擦写次数超限

**现象**：Flash 写入失败，数据丢失，MCU 运行异常。

**原因**：
- Flash 擦写寿命有限（通常 10,000~100,000 次）
- 频繁更新配置参数在同一地址
- 未做磨损均衡（Wear Leveling）

**解决方案**：
- 频繁更新的数据使用 EEPROM（1,000,000 次）或 FRAM（无限次）
- Flash 分区：将频繁更新的数据分散到多个扇区
- 实现磨损均衡算法（环形缓冲区方式写 Flash）
- 使用键值存储库（如 FlashDB / EasyFlash）

### 7.2 EEPROM 写入时间不足

**现象**：连续写 EEPROM 后数据丢失或损坏。

**原因**：
- EEPROM 单页写入需要 5~10ms 完成内部编程
- 连续写入未等待写入完成
- 页写入跨页（AT24C02 每页 8 字节），跨页部分被覆盖

**解决方案**：
- 每次页写入后延时 10ms 或轮询 ACK（设备忙时不响应）
- 单次写入不超过页大小（AT24C02: 8B, AT24C256: 64B）
- 跨页写入需分段，每段间等待
- 使用 EEPROM 状态寄存器或 ACK 轮询判断写入完成

### 7.3 SD 卡兼容性差

**现象**：某些 SD 卡无法初始化，或读写偶发失败。

**原因**：
- 不同 SD 卡厂商初始化时序有差异
- SD 卡供电不足（SD 卡瞬时电流可达 200mA）
- SPI 模式时钟频率设置不当
- 文件系统库不兼容

**解决方案**：
- 初始化阶段用 400kHz 低速时钟，初始化成功后提速
- SD 卡供电用独立 LDO（如 RT9013），输出电容 ≥ 10μF
- 增加 SD 卡检测和重试机制（5 次重试）
- 使用 FatFs 库，适配多种 SD/MMC 卡
- 测试至少 3 个不同品牌 SD 卡

## 8. EMC 类问题

### 8.1 辐射超标

**现象**：产品 EMC 测试辐射超标，附近电子设备受干扰。

**原因**：
- 高频时钟信号（MCU 主频、SPI、I2C）辐射
- DC-DC 开关频率及其谐波辐射
- PCB 回流环路面积过大
- 线缆充当天线

**解决方案**：
- 高频信号走线下方保持完整地平面
- 时钟线加 22~100Ω 串联电阻减缓边沿
- DC-DC 输入输出加 LC 滤波，电感选屏蔽型
- 信号线远离板边，线缆加磁环
- 关键信号走内层（4 层板以上）

### 8.2 静电死机

**现象**：接触金属外壳或按键时 MCU 死机或复位。

**原因**：
- 外露接口（USB、按键、天线）无 ESD 保护
- ESD 能量通过 PCB 传导到 MCU
- 地线阻抗高，ESD 电流引起地弹

**解决方案**：
- 所有外露接口加 ESD 保护器件（TVS 二极管）
- ESD 器件放在连接器最前端，泄放路径直接到地
- 按键加 100nF 到地 + 限流电阻
- PCB 地平面完整，降低地阻抗
- 金属外壳接大地

### 8.3 电源干扰导致复位

**现象**：继电器/电机动作时 MCU 复位，或 ADC 读数跳变。

**原因**：
- 继电器/电机线圈断电时产生反向电动势（可达数百伏）
- 大电流切换导致电源电压跌落
- 电机 PWM 开关噪声耦合到电源

**解决方案**：
- 继电器线圈并联续流二极管（1N4148 或 FR107）
- 电机驱动用独立电源，与 MCU 电源隔离
- MCU 电源加 TVS 保护
- 光耦隔离电机驱动信号
- 大电容（100μF~1000μF）在电源入口储能

## 9. 低功耗类问题

### 9.1 休眠电流远超规格

**现象**：MCU 进入休眠模式后，电流仍然数 mA 级别（预期 < 10μA）。

**原因**：
- 未使用的外设时钟未关闭
- GPIO 未配置为模拟模式，浮空输入产生漏电流
- 外部上拉/下拉电阻持续消耗电流
- 外部器件未进入低功耗模式
- LDO 静态电流偏大
- RTC/看门狗仍以高速时钟运行

**解决方案**：
- 休眠前逐个关闭外设时钟（USART/SPI/I2C/ADC/TIM）
- 未使用 GPIO 配置为模拟输入（Analog mode）
- 检查每个 GPIO 的外部电路，确保无持续电流路径
- 外部传感器发送睡眠命令（大部分传感器支持低功耗模式）
- 选用低静态电流 LDO（如 HT7333: 4μA, ME6211: 3μA）
- 降低 RTC 时钟频率，关闭不必要的中断源
- 调试方法：逐一断开外设电源，用万用表 μA 档测量

### 9.2 GPIO 漏电流

**现象**：休眠电流偏高，排查发现某 GPIO 在持续消耗电流。

**原因**：
- GPIO 配置为推挽输出高/低，但外部有低/高电平拉拽
- GPIO 配置为浮空输入，引脚电平不确定，输入缓冲器翻转消耗电流
- 上拉/下拉电阻与外部电路形成电流回路
- GPIO 驱动外部 LED 或其他负载未关闭

**解决方案**：
- 休眠前逐个检查 GPIO 状态
- 推挽输出引脚：如果外部有不同电平，改为模拟模式
- 浮空输入引脚：改为模拟模式或上拉/下拉
- 有外部上拉的引脚：配置为输入上拉（无电平差 = 无电流）
- LED 驱动引脚：配置为模拟模式或输出低（看电路拓扑）

### 9.3 外设未关闭

**现象**：MCU 进入 Stop 模式，但 ADC/UART 等外设仍消耗电流。

**原因**：
- 进入低功耗模式前未调用外设 DeInit/Disable
- ADC 使能位未清除
- UART 接收使能未关闭，持续监听消耗电流
- 定时器未关闭

**解决方案**：
```c
/* 进入低功耗前的标准操作 */
void enter_low_power(void) {
    /* 1. 关闭 ADC */
    ADC->CR2 &= ~ADC_CR2_ADON;
    /* 2. 关闭 UART */
    USART1->CR1 &= ~(USART_CR1_UE | USART_CR1_RE);
    /* 3. 关闭 SPI */
    SPI1->CR1 &= ~SPI_CR1_SPE;
    /* 4. 关闭不用的定时器 */
    TIM2->CR1 &= ~TIM_CR1_CEN;
    /* 5. 关闭外设时钟 */
    RCC->APB2ENR &= ~(RCC_APB2ENR_ADC1EN | RCC_APB2ENR_USART1EN);
    /* 6. GPIO 配置为模拟模式 */
    /* 7. 进入 Stop 模式 */
    PWR->CR |= PWR_CR_LPDS;  /* 低功耗 Stop */
    SCB->SCR |= SCB_SCR_SLEEPDEEP_Msk;
    __WFI();
}
```

### 9.4 唤醒后功能异常

**现象**：MCU 从低功耗唤醒后，外设不工作或数据错误。

**原因**：
- 唤醒后系统时钟切换为内部 RC（Stop 模式会停止 HSE）
- 外设寄存器在低功耗模式下被复位
- 唤醒后未重新初始化外设
- 唤醒延时不足，外设未就绪

**解决方案**：
- 唤醒后重新配置时钟树（恢复 HSE/PLL）
- 重新初始化需要使用的外设
- 唤醒后延时 10~50ms 等待外设稳定
- 测量唤醒时间：从 WFI 到第一条指令执行的时间
- Standby 模式唤醒相当于复位，所有寄存器恢复默认值
