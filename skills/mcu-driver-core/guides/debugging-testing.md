# 调试与测试规范

## 1. 硬件调试流程

### 1.1 上电前检查

**目视检查**：
- [ ] 元件极性正确 (电解电容、二极管、LED、IC 方向)
- [ ] 焊接质量 (虚焊、连锡、锡珠)
- [ ] 元件值与 BOM 一致 (电阻容值、电容耐压)
- [ ] 连接器引脚定义与原理图一致
- [ ] 无异物 (锡珠、焊锡丝残留)

**万用表检查**：
- [ ] 电源对地电阻 (无短路): 红表笔接 VCC, 黑表笔接 GND
  - 3.3V 轨: 通常 > 100Ω (具体看 IC 数量)
  - 5V 轨: 通常 > 50Ω
  - 若 < 10Ω, 可能存在短路, 禁止上电
- [ ] 各电源轨之间无短路 (VCC vs GND, 3.3V vs 5V)
- [ ] 晶振引脚对地无短路
- [ ] 关键 IC 电源引脚对地电阻正常

### 1.2 上电检测

**步骤 1: 电流检测**
```
1. 将万用表调至电流档 (mA), 串联在电源回路中
2. 限流 100mA (使用可调电源或串联电阻)
3. 上电观察电流:
   - 正常待机: 5~50mA (视 MCU 和外设而定)
   - 异常大电流 (>200mA): 立即断电, 检查短路
   - 零电流: 电源未接通或断路
```

**步骤 2: 电压检测**
- [ ] 各电源轨电压在标称值 ±5% 范围内
- [ ] LDO 输入输出压差满足 Dropout 要求
- [ ] 模拟电源 (VDDA) 电压稳定, 纹波 < 50mV
- [ ] 复位脚电平为高 (3.3V)
- [ ] 晶振引脚有电压 (约 VCC/2)

**步骤 3: 时钟检测**
- 用示波器 (10X 探头) 测晶振引脚, 确认起振
- 频率与标称值一致 (如 8MHz, 16MHz, 32.768kHz)
- 幅值 > 1V (峰峰值)
- 内部 RC 时钟可跳过此步

### 1.3 功能验证

按模块逐个验证, 遵循"从简单到复杂"原则：

```
验证顺序:
  1. GPIO 输出 (LED 闪烁)
  2. 串口打印 (UART debug log)
  3. 定时器/延时精度
  4. I2C/SPI 总线扫描
  5. 各传感器数据读取
  6. 执行器控制 (电机/舵机/继电器)
  7. 通信模块 (WiFi/蓝牙/LoRa)
  8. 完整功能联调
```

## 2. 常用调试工具使用

### 2.1 万用表

| 测量项 | 档位 | 注意事项 |
|--------|------|----------|
| 电压 | DC V | 黑表笔接地, 红表笔接测量点 |
| 电流 | DC A | 串联接入回路, 注意量程 |
| 电阻 | Ω | 断电测量, 放电后测电容 |
| 通断 | 蜂鸣档 | 用于查短路和连线 |
| 二极管 | 二极管档 | 红正黑负, 测正向压降 |

### 2.2 示波器

**基本设置**：
- 探头：10X (减小寄生电容对电路影响)
- 接地夹：就近接地, 接地线尽量短
- 耦合：DC (一般), AC (测纹波去直流分量)
- 触发：边沿触发, 上升沿, 自动触发

**典型测量场景**：

| 场景 | 设置 | 关注点 |
|------|------|--------|
| 电源纹波 | AC 耦合, 20MHz 带宽限制 | 峰峰值 < 100mV |
| I2C 信号 | DC 耦合, 上升沿触发 | 电平幅值, 上升时间, 时序 |
| SPI 信号 | DC 耦合, CS 下降沿触发 | 时钟频率, 建立保持时间 |
| UART 波形 | DC 耦合, 下降沿触发 | 波特率, 电平幅值 |
| 晶振波形 | DC 耦合, 10X 探头 | 频率, 幅值 (正弦波) |
| 复位信号 | DC 耦合, 单次触发 | 上电复位波形, 防毛刺 |

### 2.3 逻辑分析仪

适用于多通道数字信号分析 (I2C, SPI, UART, CAN 等协议解码)：

**使用步骤**：
1. 连接探针到被测信号 (SCL/SDA, MOSI/MISO/SCK/CS, TX/RX)
2. 设置采样率 (至少 4 倍信号频率, 建议 10 倍)
3. 设置触发行 (如 I2C Start 条件)
4. 采集后使用协议解码功能分析

**常用协议解码参数**：

| 协议 | 解码参数 | 注意事项 |
|------|----------|----------|
| I2C | 7位地址, ACK/NACK | 注意地址是否左移 |
| SPI | CPOL/CPHA, CS 极性, 位序 | 先确定时钟模式 |
| UART | 波特率, 数据位, 校验, 停止位 | 波特率匹配是前提 |
| CAN | 11/29位 ID, 波特率 | 差分信号需差分探头 |

### 2.4 串口调试助手

**常用功能**：
- 十六进制/ASCII 显示切换
- 自动换行/时间戳
- 定时发送
- 脚本/宏发送
- 文件传输 (XMODEM/YMODEM)

**调试技巧**：
- 用 `printf` 重定向到 UART 输出调试信息
- 用不同前缀区分日志级别: `[E]` Error, `[W]` Warn, `[I]` Info, `[D]` Debug
- 关键变量实时打印, 格式统一便于解析
- 使用 ANSI 颜色码区分日志级别 (部分终端支持)

## 3. 通信协议调试方法

### 3.1 I2C 调试

```
调试步骤:
  1. 确认上拉电阻 (4.7k~10kΩ, 用万用表测 SCL/SDA 对 VCC 阻值)
  2. 总线扫描: 逐地址发读请求, 看哪个 ACK
  3. 示波器检查:
     - Start 条件: SDA 下降沿, SCL 高
     - Stop 条件: SDA 上升沿, SCL 高
     - 电平幅值: 接近 VCC (上拉充分)
     - 上升时间: < 1μs (100kHz), < 300ns (400kHz)
  4. 逻辑分析仪解码: 验证地址和数据是否正确
  5. 常见问题:
     - 无响应: 地址错误 / 上拉缺失 / 接线错误
     - NACK: 器件未就绪 / 地址冲突
     - 数据错误: 时钟太快 / 线路干扰 / CRC 未通过
```

### 3.2 SPI 调试

```
调试步骤:
  1. 确认 CS 引脚控制: 传输前拉低, 传输后拉高
  2. 确认时钟模式 (CPOL/CPHA): 查器件手册
  3. 示波器检查:
     - SCK 空闲电平 = CPOL
     - 数据在采样沿稳定
     - CS 低电平期间 SCK 活跃
  4. 逻辑分析仪解码: 验证发送和接收数据
  5. 常见问题:
     - 全零/全FF: 时钟模式错误 / CS 未拉低
     - 数据移位: CPHA 设置错误
     - 通信不稳定: 时钟太快 / 走线太长
```

### 3.3 UART 调试

```
调试步骤:
  1. 确认 TX/RX 交叉连接: MCU TX -> 对端 RX, MCU RX -> 对端 TX
  2. 确认共地
  3. 确认波特率一致 (误差 < 2.5%)
  4. 示波器测 TX 引脚:
     - 空闲电平为高
     - 起始位低电平
     - 数据位 + 停止位
  5. 串口助手发送/接收测试
  6. 常见问题:
     - 乱码: 波特率不匹配 / 时钟精度不够
     - 只发不收 / 只收不发: TX/RX 接反 / 共地缺失
     - 偶尔丢数据: 缓冲区溢出 / 中断优先级低
```

### 3.4 CAN 调试

```
调试步骤:
  1. 确认终端电阻: 总线两端各 120Ω, 总线阻抗 ~60Ω
  2. 确认波特率一致
  3. 示波器 (差分探头) 测 CAN_H - CAN_L:
     - 隐性 (空闲): ~0V 差分
     - 显性 (发送): ~2V 差分
  4. 逻辑分析仪解码 CAN 帧
  5. 常见问题:
     - 无通信: 终端电阻缺失 / 节点波特率不匹配
     - 错误帧: 节点异常 / 总线干扰
     - 只有一个节点: 需至少 2 个节点才能通信
```

## 4. 驱动测试用例编写规范

### 4.1 测试用例模板

```c
/* 测试用例: I2C 传感器读取 */
typedef struct {
    const char *name;        /* 测试名称 */
    bool (*func)(void);      /* 测试函数, 返回 true=通过 */
} test_case_t;

/* TC-001: 设备存在性测试 */
bool test_device_present(void) {
    uint8_t id;
    if (i2c_reg_read(&dev, REG_CHIP_ID, &id) != I2C_OK)
        return false;
    return id == EXPECTED_CHIP_ID;
}

/* TC-002: 寄存器读写回环测试 */
bool test_register_loopback(void) {
    uint8_t w = 0xA5, r;
    if (i2c_reg_write(&dev, REG_TEST, w) != I2C_OK) return false;
    if (i2c_reg_read(&dev, REG_TEST, &r) != I2C_OK) return false;
    return r == w;
}

/* TC-003: 数据有效性测试 */
bool test_data_valid(void) {
    float temp;
    for (int i = 0; i < 10; i++) {
        if (sensor_read(&temp) != SENSOR_OK) return false;
        if (temp < -40 || temp > 85) return false;  /* 合理范围检查 */
        delay_ms(100);
    }
    return true;
}

/* TC-004: CRC 校验测试 */
bool test_crc(void) {
    /* 故意翻转一位, 确认 CRC 检测到错误 */
    uint8_t data[6] = {0x00, 0x00, 0x00, 0x00, 0x00, 0x00};
    data[0] ^= 0x01;  /* 翻转 bit0 */
    uint8_t crc = i2c_crc8(data, 2);
    return crc != data[2];  /* CRC 应不匹配 */
}

/* 测试用例表 */
static const test_case_t tests[] = {
    {"Device Present",      test_device_present},
    {"Register Loopback",   test_register_loopback},
    {"Data Valid",          test_data_valid},
    {"CRC Verification",    test_crc},
};

/* 测试执行框架 */
void run_tests(void) {
    int pass = 0, total = sizeof(tests) / sizeof(tests[0]);
    for (int i = 0; i < total; i++) {
        bool ok = tests[i].func();
        printf("[TC-%03d] %-30s %s\n", i+1, tests[i].name,
               ok ? "PASS" : "FAIL");
        if (ok) pass++;
    }
    printf("Result: %d/%d passed\n", pass, total);
}
```

### 4.2 测试覆盖要求

| 测试类型 | 覆盖内容 | 示例 |
|----------|----------|------|
| 功能测试 | 正常操作路径 | 读温度、写配置 |
| 边界测试 | 极限参数 | 最低/最高温度, 最大采样率 |
| 异常测试 | 错误处理 | 断线、地址错误、CRC 错误 |
| 兼容测试 | 不同器件版本 | 同系列不同型号 |
| 性能测试 | 时序和吞吐 | 读取延迟, 最大采样率 |
| 稳定性测试 | 长时间运行 | 连续 24h 读数无异常 |

## 5. 压力测试和边界测试方法

### 5.1 压力测试

```
1. 高频读写测试:
   - 以最大允许频率连续读取传感器 10000 次
   - 统计成功率、平均延迟、最大延迟
   - 检查是否有偶发 CRC 错误

2. 多设备总线测试:
   - 同一 I2C/SPI 总线接多个设备
   - 快速切换设备访问, 验证 CS/地址切换无冲突

3. 电源波动测试:
   - 在额定电压 ±10% 范围内测试
   - 模拟电池电压下降 (4.2V -> 3.0V) 时设备行为

4. 长时间稳定性:
   - 连续运行 24~72 小时
   - 每秒记录数据, 检查是否有异常跳变
   - 监控内存使用 (堆栈溢出 / 内存泄漏)
```

### 5.2 边界测试

```
1. 温度边界:
   - 传感器最低/最高工作温度
   - 温度变化率测试 (快速冷热交替)

2. 频率边界:
   - I2C: 10kHz (最低) ~ 400kHz/1MHz (最高)
   - SPI: 100kHz ~ 最大支持频率
   - UART: 9600 ~ 921600 波特率

3. 数据边界:
   - 寄存器写入 0x00 和 0xFF
   - 传感器输出满量程 (最大/最小值)
   - 数据缓冲区满/空状态

4. 电压边界:
   - 最低工作电压 (欠压复位测试)
   - 最高工作电压 (过压保护测试)
   - 电源毛刺 (模拟接触不良)
```

## 6. 低功耗测试方法

### 6.1 测试步骤

```
1. 基线测量:
   - 正常运行模式电流 (Run mode)
   - 记录各外设开启/关闭时的电流

2. 休眠模式测试:
   - Sleep / Stop / Standby 模式电流
   - 唤醒时间测量 (GPIO 中断 / 定时器 / RTC)
   - 唤醒后功能恢复正常

3. 外设功耗分析:
   - 逐一关闭外设, 测量电流变化
   - 找出异常耗电的外设
   - 未使用引脚配置为模拟模式

4. 唤醒源测试:
   - GPIO 外部中断唤醒
   - RTC 定时唤醒
   - UART 接收唤醒 (部分 MCU 支持)
   - 测量唤醒到正常运行的耗时

5. 漏电流排查:
   - 逐一断开外设电源, 测量电流变化
   - 检查上拉/下拉电阻是否有持续电流路径
   - 检查 IO 是否有驱动外部负载
```

### 6.2 功耗优化检查清单

- [ ] 未使用 GPIO 配置为模拟输入
- [ ] 未使用外设时钟已关闭
- [ ] ADC 采样完成后关闭 ADC
- [ ] 通信总线空闲时关闭 (或使用低功耗模式)
- [ ] 定时器按需开启, 不用时关闭
- [ ] Flash 读取在低功耗时关闭预取
- [ ] 合理设置系统时钟频率 (够用即可, 不盲目用高频)
- [ ] 外部上拉/下拉电阻值足够大 (减少电流)
- [ ] LED 指示灯使用 PWM 调光或低占空比闪烁

## 8. 代码审查清单

### 8.1 初始化
- [ ] 所有外设初始化前检查返回值
- [ ] GPIO 方向和初始电平配置正确
- [ ] 中断优先级配置合理
- [ ] 时钟树配置正确 (各外设时钟已使能)

### 8.2 通信协议
- [ ] 所有 I2C/SPI/UART 操作有超时保护
- [ ] 接收数据有长度和有效性检查
- [ ] CRC/校验和验证已实现
- [ ] 通信错误有重试机制

### 8.3 中断处理
- [ ] ISR 中无耗时操作 (delay, printf, 大循环)
- [ ] ISR 中仅设置标志, 主循环处理逻辑
- [ ] 共享变量使用 volatile 修饰
- [ ] 中断嵌套优先级配置正确

### 8.4 内存安全
- [ ] 数组访问有边界检查
- [ ] 无未初始化变量使用
- [ ] 动态内存分配最小化 (嵌入式建议不用 malloc)
- [ ] 栈大小足够 (递归深度、局部数组)

### 8.5 错误处理
- [ ] 所有硬件操作检查返回值
- [ ] 错误状态有日志输出
- [ ] 故障恢复策略已实现 (重启外设、重置总线)
- [ ] 看门狗已启用

### 8.6 低功耗
- [ ] 休眠前关闭不必要外设
- [ ] 唤醒后重新初始化外设
- [ ] 定时器/RTC 唤醒配置正确
- [ ] 休眠模式电流符合预期
